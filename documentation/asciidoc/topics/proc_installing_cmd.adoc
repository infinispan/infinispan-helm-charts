= Installing the {brandname} chart on the command line

[role="_abstract"]
Use the command line to deploy {brandname} clusters on {openshiftshort}.

.Prerequisites

* Install the `helm` client.
* Add the link:{helm_chart_repo_readme}[{openshiftshort} Helm Charts repository].
ifdef::community[]
* Have access to an {openshiftshort} or Kubernetes cluster.
* Have a `kubectl` or `oc` client.
endif::community[]
ifdef::downstream[]
* Have access to an {openshiftshort} cluster.
* Have an `oc` client.
endif::downstream[]


.Procedure

. Create a values file for your {brandname} cluster.
+
For example, the following values file creates a cluster with two nodes:
+
[source,options="nowrap",subs=attributes+]
----
$ cat > cluster-configuration.yaml<<EOF
include::yaml/infinispan-values.yaml[]
EOF
----
+
ifdef::community[]
[TIP]
====
You can find descriptions and values for each field in the link:{helm_chart_readme}[{brandname} chart README].
====
+
endif::community[]
. Install the {brandname} chart and specify the values file for your cluster.
+
ifdef::community[]
[source,options="nowrap",subs=attributes+]
----
$ helm install infinispan openshift-helm-charts/infinispan-infinispan -f cluster-configuration.yaml
----
endif::community[]
ifdef::downstream[]
[source,options="nowrap",subs=attributes+]
----
$ helm install infinispan redhat-charts/datagrid -f cluster-configuration.yaml
----
endif::downstream[]

.Verification

Watch the pods to ensure all nodes in the {brandname} cluster are created successfully.

[source,options="nowrap",subs=attributes+]
----
$ oc get pods -w
----
